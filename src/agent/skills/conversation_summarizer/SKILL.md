# 对话总结与知识管理 Skill

## 适用对象

**仅限助教 Agent 使用**

## 何时使用此 Skill

当用户提出以下需求时，必须调用此 Skill：

- "帮我总结今天的对话"
- "整理一下我们的聊天记录"
- "把这段对话保存到知识库"
- "提取今天关于XXX的讨论"
- "帮我归纳学习内容"
- "备份今天的对话"

**禁止直接回答**，必须先调用此 Skill 的完整流程。

## 重要提醒

**不要重复调用 download_skill！**

- 一旦开始执行此skill，所有步骤信息都保存在对话上下文中
- 按照下面的步骤顺序执行，直到任务完成
- 如果用户回复了步骤1的问题，直接进入步骤2，不要重新读取skill
- 任务完成后，可以正常回答用户的其他问题

# 详细步骤

## 步骤1: 需求确认（单次确认）

**目标**：一次性确认所有需求，避免反复询问。

**询问模板**：
```
我来帮您总结对话内容。请确认：

1. **时间范围**（必填）：
   - 今天的对话
   - 最近3天
   - 最近一周
   - 其他：____

2. **保存到知识库**（可选）：
   - 是，保存
   - 否，仅查看

请回复您的选择，例如："今天，保存" 或 "最近一周，不保存"
```

**注意**：只询问这一次，后续不再询问用户意见！

---

## 步骤2: 查询对话

**工具调用**：
```yaml
tool: assistant_conversation_conversation_query
parameters:
  start_date: "YYYY-MM-DD"
  end_date: "YYYY-MM-DD"
  limit: 100
```

**处理逻辑**：
1. 调用工具获取对话列表
2. 如果没有对话 → 告知用户并结束
3. 如果有对话 → 自动选择全部，进入步骤3

**注意**：不需要询问用户选择哪些对话，自动选择全部！

---

## 步骤3: 提取内容

**工具调用**：
```yaml
tool: assistant_conversation_conversation_get_detail
parameters:
  conversation_id: "conv_uuid"
```

**处理逻辑**：
1. 批量获取所有对话详情
2. 合并内容按时间顺序

---

# 错误处理

## 场景1：没有对话记录

```
抱歉，在指定时间范围内没有找到与教授Agent的对话记录。

可能原因：
- 该时间段没有进行对话
- 对话已被删除

建议：检查时间范围或尝试扩大查询范围
```

## 场景2：API调用失败

```
抱歉，获取对话数据时出现问题。

错误信息：[具体错误]

建议：稍后重试或联系管理员
```

# 核心原则

## DO ✅

- **只询问一次**：步骤1一次性确认所有需求
- **自动执行**：后续步骤自动完成，不再询问
- **直接交付**：生成结果后直接展示或保存
- **简洁高效**：避免反复确认和循环询问

## DON'T ❌

- 不要反复询问用户意见
- 不要询问"是否满意"
- 不要二次确认保存
- 不要询问"选择哪些对话"（自动全选）
