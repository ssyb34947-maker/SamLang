# 资料整理 Skill

## 使用场景

当用户需要以下帮助时，使用此 Skill：

- "帮我总结今天的学习内容"
- "整理一下我们的对话"
- "把这段对话加入知识库"
- "知识库里有哪些文档？"
- "删除知识库里的某个文档"
- "重命名某个对话"

## 核心工作流程：将对话保存到知识库

当用户说"帮我把今天的对话存进知识库"时，执行以下**3步流程**：

### 第1步：查询今日对话
**工具**: `conversation_query`
- 参数: start_date=今天日期, end_date=今天日期
- 获取今日所有对话列表
- 向用户确认要整理哪些对话

### 第2步：获取对话内容并总结
**工具**: `conversation_get_detail`
- 对选中的对话，获取完整消息内容
- AI自主分析并生成学习总结
- 总结应包含：
  - 学习的主题/知识点
  - 关键概念和解释
  - 重要的代码片段或示例
  - 待办事项或后续学习计划

### 第3步：保存到知识库
**工具**: `knowledge_add_text`
- 参数:
  - title: 文档标题（如"2024-01-15 学习总结"）
  - content: 总结内容
  - doc_type: "note"（笔记类型）
  - confirmed: false（先获取用户确认）
- **必须先确认**: 向用户展示标题和内容预览，确认后再设置 confirmed=true 执行保存

**完整示例**:
```
用户: 帮我把今天的对话存进知识库

助手: 我来帮您整理今天的对话并保存到知识库。

[第1步] 查询今日对话...
找到3个对话：
1. Python装饰器学习
2. 数据库优化讨论
3. API设计规范

您希望整理哪些对话？（默认全部）

[第2步] 获取对话内容并总结...
根据今天的对话，我为您整理了以下学习内容：

## Python装饰器
- 学习了@decorator的语法和原理
- 掌握了装饰器在日志记录中的应用

## 数据库优化
- 索引设计的最佳实践
- 查询性能分析技巧

## API设计
- RESTful规范要点
- 版本控制策略

[第3步] 保存到知识库
即将保存以下内容到知识库：
  标题: 2024-01-15 学习总结
  类型: note
  内容长度: 1250 字符
  内容预览: ## Python装饰器...

请确认：是否保存到知识库？

用户: 确认

助手: [调用 knowledge_add_text，confirmed=true]
✅ 内容已成功保存到知识库！
```

## 其他工作流程

### 管理知识库文档

#### 查看知识库列表
- 使用 `knowledge_list_sources`
- 显示所有源文档（chunk前的原始文档）

#### 上传文件到知识库
- 使用 `knowledge_upload`（上传本地文件）
- **必须先确认**: 上传前向用户确认文件路径和类型
- 支持的格式: PDF, Word, Excel, CSV, 文本, JSON, 图片

#### 直接保存文本到知识库
- 使用 `knowledge_add_text`（保存AI生成的内容）
- **必须先确认**: 向用户确认标题和内容
- 适用于保存学习总结、笔记等

#### 删除知识库文档
- 使用 `knowledge_delete`
- **必须先确认**: 删除前向用户确认，此操作不可恢复！
- 需要文档ID（可从列表中获取）

#### 查看文档详情
- 使用 `knowledge_get_detail`
- 查看文档的分块情况和内容预览

## 重要原则

### 1. 确认原则
以下操作**必须**先向用户确认：
- ❌ 删除对话（不可恢复）
- ❌ 删除知识库文档（不可恢复）
- ⚠️ 上传文档到知识库
- ⚠️ 重命名对话

### 2. 数据隐私
- 只能访问用户自己的对话和文档
- 系统文档只读，不可修改
- 无法访问其他用户的数据

### 3. 操作流程
1. **理解需求**：明确用户想要什么
2. **查询信息**：使用工具获取必要数据
3. **展示结果**：向用户展示查询结果
4. **确认操作**：需要修改时先确认
5. **执行操作**：确认后执行
6. **反馈结果**：告知操作结果
