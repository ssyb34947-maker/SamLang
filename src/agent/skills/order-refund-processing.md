# 订单退款处理 SOP

## 适用场景
- 用户明确要求退款
- 订单处于可退款状态（未发货、已发货但在退货期内等）
- 用户投诉产品质量、物流问题、误操作等合理退款理由
- 不适用于：已签收超过7天、个性化定制商品、明显恶意退款

## 执行步骤（按顺序执行）

1. **验证订单信息**
   - 获取订单ID（从用户消息中提取，或询问用户提供）
   - 调用工具：`mcp_get_order_detail(order_id)`
   - 检查字段：
     - order_status：必须是 "paid"、"shipped"、"delivered" 中的一种
     - refund_status：必须是 "none" 或 "pending"
     - created_at：判断是否在退款时效内（一般30天）
     - payment_amount：记录原始支付金额

2. **确认退款理由**
   - 询问用户具体退款原因（如果消息中未说明）
   - 常见分类：
     - 质量问题 → 需用户上传照片/视频（调用 upload_evidence tool）
     - 错发/漏发 → 核对订单商品
     - 后悔/不想要 → 判断是否支持无理由退货
   - 如果理由不合理，礼貌引导其他解决方案（如换货、优惠券）

3. **检查退款资格**
   - 调用工具：`mcp_check_refund_eligibility(order_id, reason)`
   - 返回结果可能包含：
     - eligible: true/false
     - refund_amount: 计算后的可退金额（可能扣除运费、手续费）
     - restrictions: 限制说明（如“仅退部分金额”）

4. **发起退款申请**
   - 如果 eligible 为 true：
     - 调用工具：`mcp_initiate_refund(order_id, amount, reason)`
     - 记录退款单号（refund_id）
   - 如果失败：
     - 记录失败原因
     - 向用户说明原因，并提供下一步建议（如联系客服、申请售后）

5. **通知用户 & 后续跟进**
   - 回复用户：
     - 退款已发起，预计到账时间（3-10个工作日，视支付渠道）
     - 退款单号：{refund_id}
     - 如有问题可回复“查询退款{refund_id}”
   - 如果涉及人工审核，告知用户“正在审核中，24小时内反馈”

6. **记录日志**
   - 调用工具：`mcp_log_refund_event(order_id, refund_id, status, operator="AI-Agent")`

## 常见异常处理
- 订单不存在 → “抱歉，我找不到这个订单，请提供正确的订单号”
- 已退款 → “该订单已于{date}完成退款，请检查支付账户”
- 不符合退款条件 → 礼貌说明原因，提供替代方案（如优惠券、积分补偿）
- 用户情绪激动 → 使用安抚话术：“非常理解您的不满，我们会尽快处理”

## 示例对话
用户：我买的那个T恤质量有问题，我想退款，订单号是ORD-202503051234

Agent：
1. 调用 mcp_get_order_detail("ORD-202503051234")
2. 确认订单存在、状态可退
3. 询问照片证据
4. 用户上传后调用 mcp_initiate_refund
5. 回复：“退款已发起，金额 ¥89.00，预计3-5个工作日到账。退款单号：REF-20250306-001”
